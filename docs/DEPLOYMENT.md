# Validation Bench — Vercel Deployment Guide

Этот документ описывает процесс публикации стенда приёмочных испытаний на платформу Vercel.

## 🚀 Пошаговое развёртывание

1. **Создание проекта в Vercel:**
   * Подключите ваш репозиторий `voice-assistant` к Vercel.
   * Установите **Framework Preset** в значение `Vite` (или `Other`, если конфигурация кастомная).
   * Укажите **Root Directory** как корень репозитория (или настройте сборку для монорепозитория, если используется `apps/voice-demo`).

2. **Команды сборки (Build Settings):**
   * **Build Command:** `npm run build` (или команда сборки вашего конкретного приложения).
   * **Output Directory:** `dist` (или папка, куда компилируется Vite-проект).

## ⚙️ Переменные окружения (Environment Variables)

В панели управления Vercel необходимо обязательно добавить следующие переменные:

* `VITE_BACKEND_API_URL` — Полный URL-адрес существующего Backend API (например, `https://ibronevik.ru`).
* `VITE_AUTH_TOKEN` — Базовый токен авторизации (если требуется на этапе инициализации).

## 🔒 Ограничения браузеров и HTTPS

* **Работа через HTTPS:** Поскольку Vercel автоматически предоставляет SSL-сертификат (HTTPS), ваше приложение будет работать по защищённому протоколу.
* **Mixed Content Warning:** Убедитесь, что `VITE_BACKEND_API_URL` также использует протокол **HTTPS**. Если делать запросы на `http://`, браузер заблокирует их из соображений безопасности.
* **CORS Policy:** На стороне Backend API должен быть разрешён доступ (CORS) для вашего домена Vercel (например, `*.vercel.app`), иначе браузер отклонит запросы `fetch`.

## 👥 Инструкция для удалённых тестировщиков (Remote Testing)

После успешного деплоя удалённому тестировщику для работы **НЕ ТРЕБУЕТСЯ** установка Node.js или клонирование репозитория. Достаточно:
1. Перейти по сгенерированной ссылке `https://your-project.vercel.app`.
2. Авторизоваться в интерфейсе (система выполнит `POST /api/v1/auth/`).
3. Запустить интерактивный или автоматический тест.
4. Проверить результаты во вкладке **Report Preview**.
5. Скачать JSON-отчёт локально или отправить его напрямую через **Send Report** (выполнит `POST /api/v1/mail/`).
